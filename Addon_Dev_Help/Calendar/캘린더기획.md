**“바탕화면 위에 항상 띄워두는 미니멀 캘린더 위젯”**입니다.
기능을 많이 보여주는 관리형 캘린더가 아니라, **한눈에 오늘 일정과 월간 흐름만 확인하는 데스크톱용 UI**에 가깝습니다.

## 전체 콘셉트

**화이트/글래스모피즘 스타일의 플로팅 캘린더 위젯**입니다.

배경은 은은한 파스텔 톤의 데스크톱 배경이고, 그 위에 반투명 흰색 카드 형태의 캘린더가 떠 있습니다.
사용자가 바탕화면을 볼 때 방해되지 않도록 색상과 정보량을 줄이고, 필요한 일정만 간단히 보여주는 구조입니다.

## 화면 구성

중앙에는 큰 월간 캘린더가 있고, 오른쪽에는 오늘 일정과 다가오는 일정만 정리된 작은 패널이 있습니다.

구성 비율은 대략 다음과 같습니다.

```text
┌──────────────────────────────────────────────┐
│  상단: 월 이동 / 현재 월 / 오늘 / + 일정       │
├──────────────────────────────┬───────────────┤
│                              │ 오늘 일정      │
│        월간 캘린더            │ 다가오는 일정   │
│                              │               │
└──────────────────────────────┴───────────────┘
```

## 주요 UI 요소

상단에는 아주 기본적인 조작만 남겼습니다.

```text
<   >     2024년 7월              오늘    + 일정
```

각 기능은 다음 의미입니다.

* `<`, `>`: 이전 달 / 다음 달 이동
* `2024년 7월`: 현재 보고 있는 월 표시
* `오늘`: 오늘 날짜로 빠르게 이동
* `+ 일정`: 새 일정 추가 버튼

복잡한 검색창, 필터, 사이드바, 프로필, 설정 메뉴는 제거되어 있습니다.

## 월간 캘린더 영역

월간 캘린더는 넓고 여백이 많은 그리드로 구성되어 있습니다.

특징은 다음과 같습니다.

* 날짜 숫자를 크게 배치
* 일요일은 붉은색 계열
* 토요일은 푸른색 계열
* 오늘 날짜는 보라색 원으로 강조
* 일정이 있는 날짜는 작은 색상 점으로 표시
* 긴 일정 제목은 날짜 칸 안에 넣지 않음

즉, 날짜 칸 내부는 최대한 비워두고, 일정 존재 여부만 색상 점으로 알려주는 방식입니다.

## 오른쪽 일정 패널

오른쪽 패널은 두 섹션으로 나뉩니다.

```text
오늘 일정
- 팀 스탠드업     09:30
- 제품 리뷰       14:00
- 디자인 검토     16:30

다가오는 일정
- 정기 회고          7.25 (목)
- 마케팅 전략 회의   7.29 (월)
- OKR 체크인         8.2 (금)
```

이 패널의 목적은 **날짜 칸 안에 긴 텍스트를 넣지 않고도 오늘과 가까운 일정만 빠르게 보여주는 것**입니다.

## 색상 사용 방식

이 UI는 색상을 많이 쓰지 않습니다.
대신 작은 점으로 일정 유형을 구분합니다.

예시는 다음과 같습니다.

* 보라색: 미팅 / 중요 일정
* 파란색: 업무 / 리뷰
* 주황색: 디자인 / 체크인 / 작업성 일정

색상은 정보 전달용으로만 사용하고, 전체 UI는 흰색과 연회색 중심으로 유지합니다.

## 바탕화면용으로 좋은 이유

이 디자인은 바탕화면에 띄워두기 적합합니다.

이유는 다음과 같습니다.

* 정보량이 적어 시야를 방해하지 않음
* 반투명 카드라 배경화면과 잘 어울림
* 일정이 많은 날도 날짜 칸이 복잡해지지 않음
* 오늘 일정은 오른쪽에서 바로 확인 가능
* `+ 일정` 버튼만으로 빠른 입력 흐름을 만들 수 있음
* 검색/필터/설정 같은 복잡한 기능은 숨길 수 있음

## 개발 시 구조 제안

웹 프론트엔드로 구현한다면 컴포넌트를 이렇게 나누면 좋습니다.

```text
DesktopCalendarWidget
├─ CalendarHeader
│  ├─ MonthNavigator
│  ├─ TodayButton
│  └─ AddEventButton
├─ MonthGrid
│  ├─ WeekdayHeader
│  └─ DayCell
│     └─ EventDots
└─ AgendaPanel
   ├─ TodayEvents
   └─ UpcomingEvents
```

이 구조면 나중에 기능을 추가해도 깔끔합니다.

## 최종 방향

이 이미지는 **“캘린더 앱 전체 화면”**이라기보다,
**“Web OS 바탕화면에 올려두는 데스크톱 캘린더 위젯”**에 가깝습니다.

따라서 핵심 방향은 다음과 같습니다.

```text
단순한 월간 보기
+ 오늘 일정
+ 다가오는 일정
+ 빠른 일정 추가
```

이 네 가지만 유지하는 것이 가장 좋습니다.
