# 웹OS (Web OS) 실행 가이드

> [!WARNING]
> **현재 개발 중인 미완성 프로젝트입니다.** 일부 기능이 제한되거나 버그가 있을 수 있습니다.

본 프로젝트는 로컬 PC 환경에서 작동하는 웹 기반 운영체제입니다. 서버와 클라이언트를 각각 실행해야 정상적으로 작동합니다. (현재 모든 필요한 패키지는 이미 설치되어 있습니다.)

## 🚀 실행 단계

### 1. 백엔드 서버 실행 (파일 제어 및 터미널 서버)
새 터미널을 열고 아래 명령어를 입력하여 서버를 실행합니다.
```bash
cd /home/inri/문서/web_os
node server/index.js
```
* 서버가 성공적으로 실행되면 `Web OS Server running on port 3000` 문구가 표시됩니다.

### 2. 프론트엔드 클라이언트 실행 (웹 인터페이스)
또 다른 새로운 터미널을 열고 아래 명령어를 입력하여 사용자 인터페이스를 실행합니다.
```bash
cd /home/inri/문서/web_os/client
npm run dev
```
* 실행 후 터미널에 나타나는 주소(보통 `http://localhost:5173`)를 클릭하거나 브라우저 주소창에 입력합니다.

## 🛠️ 주요 기능 사용법

1.  **파일 스테이션 (File Station)**: 
    *   로컬 파일 탐색 및 CRUD 지원.
    *   **인벤토리(Inventory)**: 시스템 자산(배경화면, 아이콘) 전용 보관소 지원 (`server/storage/inventory`).
    *   **지능형 경로 감지**: 리눅스(XDG), 윈도우(Known Folders) 등 환경에 맞춰 한글/영문 폴더 자동 맵핑.
2.  **터미널 (Terminal)**: 바탕화면의 터미널 아이콘을 더블 클릭하여 실제 PC 셸에 접속합니다.
3.  **리소스 모니터 (Resource Monitor)**: CPU, 메모리, 저장소, 네트워크(IP) 상태를 실시간으로 확인합니다.
4.  **설정 (Settings)**: 탭 방식의 인터페이스를 통해 시스템 및 `.env` 설정을 관리합니다.

## 🛡️ 보안 및 경로 관리 (Path Security)

본 OS는 로컬 파일 시스템에 직접 접근하므로, 허가되지 않은 경로로의 접근을 방지하는 강력한 보안 메커니즘이 적용되어 있습니다.

### 1. 허용된 루트 (`ALLOWED_ROOTS`)
- 프로젝트 루트의 `.env` 파일 내 `ALLOWED_ROOTS` 변수를 통해 접근 가능한 최상위 경로를 제한합니다.
- 예: `ALLOWED_ROOTS=["/home/inri", "/mnt/data"]` 
- 이 리스트에 포함되지 않은 상위 디렉토리(예: `/etc`, `/var`)는 절대로 접근할 수 없습니다.

### 2. 패스 가드 (Path Guard) 미들웨어
- 모든 파일 요청(`list`, `read`, `write`, `delete`)은 실행 전 `Path Guard`를 거칩니다.
- 사용자가 보낸 경로를 절대 경로로 계산한 후, 설정된 `ALLOWED_ROOTS` 중 하나로 시작하는지 검증합니다.
- 이를 통해 `../` 등을 이용한 디렉토리 트래버스(Directory Traversal) 공격을 원천 차단합니다.

### 3. 시작 경로 (`INITIAL_PATH`)
- 파일 스테이션이 처음 열릴 때 보여줄 기본 위치를 설정합니다.
- 예: `INITIAL_PATH=/home/inri`

### 4. 지능형 라이브러리 연결
- 사용자의 OS 언어나 환경에 따라 달라지는 시스템 폴더(문서, 다운로드 등)는 서버의 `userDirs` 서비스가 자동으로 감지하여 동적으로 연결합니다.

## ⚠️ 참고 사항
-   기본적으로 `/home/inri` 경로만 접근 가능하도록 설정되어 있습니다. (보안을 위한 `Path Guard` 적용)
-   설정 변경은 프로젝트 루트의 `.env` 파일 또는 `config.json`을 통해 가능합니다.

## 🚀 향후 기능 고도화 계획 (Suggestions)

본 프로젝트의 완성도를 높이기 위한 아이디어들입니다:

1.  **클라우드 동기화**: Google Drive 또는 Dropbox와 연동하여 외부 저장소 파일을 관리하는 기능.
2.  **멀티태스킹 강화**: 윈도우 스냅(Snap), 타일링 레이아웃, 가상 데스크톱 기능을 통한 작업 효율화.
3.  **앱 에코시스템**: PWA(Progressive Web App) 기술을 활용하여 웹 응용 프로그램을 "설치"하고 관리하는 대시보드.
4.  **커스텀 테마 및 위젯**: 사용자의 취향에 맞는 배경화면, 아이콘 팩, 데스크톱 위젯(날씨, 일정 등) 추가.
5.  **보안 강화**: 2단계 인증(2FA), 생체 인식 시뮬레이션, 민감 파일 암호화 저장 기능.
6.  **성능 최적화**: Web Worker를 이용한 백그라운드 처리 및 앱별 리소스 사용량 제한.
7.  **글로벌 검색**: 시스템 전체 파일 및 앱을 빠르게 찾을 수 있는 'Spotlight' 스타일의 검색창.
