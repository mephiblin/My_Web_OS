# 신규기획서: File Station Places 기본 연결 정책 (Windows/Ubuntu)

## 1. 목적

Windows/Ubuntu 설치 직후 File Station이 바로 실사용 가능한 상태가 되도록, `Places` 기본 노출 및 경로 연결 정책을 정의한다.

## 2. 핵심 원칙

- 설치 직후 `Desktop`, `Documents`, `Downloads` 접근이 가능해야 한다.
- `Home`은 Desktop이 아니라 사용자 홈 루트다.
  - Windows: `C:\Users\<user>`
  - Ubuntu: `/home/<user>`
- `Desktop`은 Home 하위의 별도 작업 표면으로 `Places`에 명시적으로 노출한다.
- 보안 경계를 유지하되, 사용자 체감상 “표시된 Places는 실제로 열려야” 한다.

## 3. 기본 Places 구성

### 3.1 Windows

- Home
- Desktop
- Documents
- Downloads
- Pictures
- Music
- Videos

### 3.2 Ubuntu

- Home
- Desktop
- Documents
- Downloads
- Pictures
- Music
- Videos

## 4. 실행 정책

- `ALLOWED_ROOTS`가 비어 있거나 레거시 제한값(프로젝트 루트 단일 경로)이면 Places 기준 기본 루트를 자동 채운다.
- `INITIAL_PATH` 기본값은 `Documents`, 없으면 `Home`으로 폴백한다.
- Places에 표시된 항목은 `/api/fs/list` 기준으로 실제 탐색 가능해야 한다.
- 표시-접근 불일치(클릭 시 `403`)는 버그로 본다.

## 5. 검증 기준

- 서버 기동 직후 `GET /api/fs/user-dirs`에서 Home/Desktop/Documents/Downloads가 유효 경로를 반환한다.
- 각 Places 경로에 대해 `GET /api/fs/list?path=...`가 성공한다.
- `Home`과 `Desktop`이 서로 다른 의미로 동작한다.

## 6. 향후 Docker 포팅 시 원칙

- 컨테이너 경로와 호스트 경로를 1:1로 착각하지 않도록 Places 라벨과 실제 루트를 명확히 분리한다.
- Docker 환경에서도 “표시된 Places는 열리는가”를 동일한 품질 기준으로 유지한다.
