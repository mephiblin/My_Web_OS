# 만들 것들

이 문서는 `My_Web_OS`의 `Package Center`와 `Sandbox` 위에서 앞으로 만들어볼 패키지, 위젯, 도구들을 정리한 작업 목록이다.

목표는 단순한 아이디어 나열이 아니라, 실제로 어떤 순서로 만들면 좋은지까지 같이 정리하는 것이다.

---

## 1. 방향성

우리가 만들고 싶은 것은 단순한 앱 모음이 아니다.

- 로컬 PC에서 바로 쓸 수 있는 도구
- 샌드박스 안에서 안전하게 돌아가는 작은 앱
- 위젯으로도 쓸 수 있는 가벼운 기능
- Package Center에서 만들고, 수정하고, 실행할 수 있는 패키지

즉, 최종 목표는 `작은 앱 생태계`다.

---

## 2. 우선순위 기준

아래 기준으로 먼저 만들 것을 고른다.

- 자주 쓸수록 우선순위가 높다.
- 구현이 너무 무겁지 않아야 한다.
- 샌드박스 구조와 잘 맞아야 한다.
- 위젯으로도 확장 가능하면 더 좋다.
- Package Center의 존재 이유를 보여줄 수 있어야 한다.

---

## 3. 1차로 만들 것들

### 3.1 메모 앱

가장 먼저 만들기 좋은 패키지다.

- 간단한 텍스트 메모
- 마크다운 작성
- 자동 저장
- 검색
- 날짜별 정리

왜 먼저 만드나:
- 만들기 쉽다.
- 실사용성이 높다.
- 샌드박스 데이터 저장 흐름을 시험하기 좋다.

형태:
- 앱
- 나중에는 위젯도 가능

---

### 3.2 할 일 앱

간단한 To-do / Task 패키지다.

- 할 일 추가
- 체크 완료
- 우선순위
- 오늘 할 일 / 이번 주 할 일
- 완료 항목 숨기기

왜 좋나:
- 샌드박스 앱 상태 저장 연습에 좋다.
- 위젯으로도 매우 잘 맞는다.

형태:
- 앱
- 위젯
- 하이브리드 추천

---

### 3.3 북마크 관리자

자주 가는 링크를 모으는 앱이다.

- 폴더별 북마크
- 태그
- 최근 열기
- 링크 검색
- 외부 링크 빠른 실행

왜 좋나:
- 로컬 웹OS와 매우 잘 맞는다.
- 관리 콘솔 느낌이 강하다.

형태:
- 앱
- 위젯은 최근 북마크만 표시하는 방식 가능

---

### 3.4 클립보드 히스토리

복사한 텍스트를 저장하는 패키지다.

- 복사 기록
- 다시 붙여넣기
- 즐겨찾기
- 검색

왜 좋나:
- 매우 자주 쓰인다.
- 작은 기능이지만 만족도가 높다.

형태:
- 앱
- 위젯은 최근 항목 몇 개만 보여주는 식으로 가능

---

### 3.5 계산기

가장 단순한 유틸리티 앱이다.

- 사칙연산
- 메모리 기능
- 간단한 단위 변환

왜 좋나:
- 샌드박스 앱의 최소 예시로 좋다.
- 테스트용으로도 좋다.

형태:
- 앱
- 위젯 가능

---

## 4. 시놀로지 느낌이 나는 것들

### 4.1 Download Station

다운로드를 관리하는 패키지다.

- 다운로드 목록
- 진행률
- 완료/실패 상태
- 파일 위치 열기
- 일시정지/재개

왜 좋나:
- 시놀로지 느낌이 가장 강한 패키지 중 하나다.
- 네트워크/파일/상태 관리가 모두 들어간다.

형태:
- 앱

---

### 4.2 Photo Station

이미지 관리 앱이다.

- 폴더별 앨범
- 썸네일 보기
- 슬라이드쇼
- 날짜별 정렬
- 태그

왜 좋나:
- 미디어 뷰어와 자연스럽게 연결된다.
- 로컬 사진 정리에 실용적이다.

형태:
- 앱
- 위젯은 최근 사진 몇 장 미리보기 가능

---

### 4.3 Music Station

음악 플레이어 겸 라이브러리다.

- 앨범/아티스트/곡 보기
- 재생목록
- 최근 재생
- 간단한 플레이어 컨트롤

왜 좋나:
- 미디어 앱 생태계를 넓힌다.
- 기존 media viewer와 역할을 분리할 수 있다.

형태:
- 앱
- 위젯은 플레이어 컨트롤 가능

---

### 4.4 Video Station

영상 관리 앱이다.

- 영상 목록
- 이어보기
- 자막 연결
- 썸네일

왜 좋나:
- 시놀로지 느낌이 아주 강하다.
- 기존 미디어 모듈과 시너지가 있다.

형태:
- 앱

---

### 4.5 Document Station

문서 모음 관리 앱이다.

- PDF
- TXT
- MD
- 문서 폴더별 정리

왜 좋나:
- 파일 스테이션과 문서 뷰어 사이를 정리해준다.

형태:
- 앱

---

## 5. 홈랩 / NAS 느낌 패키지

### 5.1 서버 모니터

시스템 상태를 보는 패키지다.

- CPU
- 메모리
- 디스크
- 네트워크
- 온도

왜 좋나:
- 웹OS의 존재감을 보여준다.
- 위젯으로도 매우 좋다.

형태:
- 앱
- 위젯
- 하이브리드 추천

---

### 5.2 로그 분석기

로그를 탐색하는 패키지다.

- 로그 파일 열기
- 검색
- 필터
- 시간대별 보기

왜 좋나:
- 현재 로그 센터를 더 실용적으로 만든다.

형태:
- 앱

---

### 5.3 백업 관리자

백업 작업을 묶어서 관리하는 패키지다.

- 백업 작업 목록
- 마지막 실행 시각
- 실패/성공 상태
- 복구 지점

왜 좋나:
- 홈랩/NAS 분위기가 강하다.

형태:
- 앱

---

### 5.4 서비스 대시보드

내부 서비스 상태를 보여주는 패키지다.

- 서비스 목록
- 상태
- 재시작
- 에러 표시

왜 좋나:
- `/api/services`와 바로 연결된다.
- 운영자용 느낌이 강하다.

형태:
- 앱

형태:

형태:

## 7. 위젯으로 특히 좋은 것들

아래는 처음부터 위젯으로 만들면 좋은 것들이다.

- 시계
- 날씨
- CPU 사용률
- RAM 사용률
- 디스크 사용량
- 네트워크 트래픽
- 오늘 할 일
- 최근 메모
- 최근 북마크
- 음악 재생 컨트롤

위젯은 작고 단순할수록 좋다.

---

## 8. 하이브리드로 만들면 좋은 것들

아래는 앱이면서 위젯이기도 한 형태가 가장 자연스럽다.

- 시계
- 할 일
- 메모
- 서버 모니터
- 날씨
- 음악 컨트롤
- 북마크

이런 것들은 하나의 패키지로 만들고,

- 앱 모드
- 위젯 모드

둘 다 지원하는 방향이 좋다.

---

## 9. Python으로 해볼 수 있는 것들

Python은 우선순위가 조금 낮지만, 나중에 꽤 재미있는 패키지가 가능하다.

- 간단한 데이터 분석 도구
- CSV 뷰어
- 텍스트 처리기
- 간단한 계산 도구
- 교육용 샘플 앱

단, Python은 처음부터 서버 실행형으로 가기보다,

- 브라우저형 실행
- 제한된 권한
- 실험용 템플릿

순서가 더 적합하다.

## 11. Package Center에서 구현해야 할 기능

만들 것들을 실제로 패키지 센터에서 다루려면 아래가 필요하다.

- 새 패키지 생성
- 템플릿 선택
- manifest 편집
- 파일 트리 편집
- 저장
- 미리보기
- 실행
- 복제
- 내보내기
- 삭제

즉, 패키지 센터는 목록만 보여주는 곳이 아니라

**앱을 만드는 작업대**

가 되어야 한다.

## 13. 현실적인 개발 원칙

- 처음부터 거대한 앱을 만들지 않는다.
- 작고 자주 쓰는 앱부터 만든다.
- 앱과 위젯은 가능하면 같은 구조로 설계한다.
- 권한은 최소로 시작한다.
- 샌드박스 밖으로 나가는 기능은 천천히 연다.

---

## 14. 최종 정리

우리가 만들 만한 것들은 크게 네 부류다.

- 일상 도구
- 시놀로지 느낌의 관리 앱
- 개발자용 도구
- 위젯/하이브리드 패키지

가장 좋은 방향은,

**자주 쓰는 작은 도구부터 만들고, 그다음 시놀로지 같은 관리형 패키지로 확장하고, 마지막에 앱/위젯/Python을 하나의 패키지 체계로 묶는 것**

이다.
